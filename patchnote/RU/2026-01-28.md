# Детали проектов, статистика GitHub и отслеживание активности

Это обновление заметно повышает уровень портфолио: улучшенная навигация, детальные страницы проектов и более точная аналитика активности GitHub.

Дата: 28 января 2026

## Карточки проектов → страницы деталей

- Добавлена кнопка **«Посмотреть детали проекта»** на каждой карточке проекта.
- Эта кнопка ведёт на **отдельную страницу конкретного проекта**.
- Плавный, современный интерфейс с иконками и понятной визуальной иерархией.
- Вся важная информация о проекте собрана на одной странице.

> Реальный пример: вместо того чтобы переключаться между сайтом, GitHub и внешними статистиками, у тебя всё перед глазами — в одном месте.

---

## Страница статистики GitHub

Новая страница, посвящённая анализу GitHub для проекта.

### Выбор репозитория

- Обязательный выбор GitHub-репозитория.
- Нельзя показать статистику, пока не выбран хотя бы один репозиторий (это предотвращает странные «пустые» состояния).

### Общая статистика

- Количество коммитов
- Количество участников (contributor'ов)
- Добавленные / удалённые строки
- Звёзды (stars)
- Форки (forks)
- Размер репозитория
- Дата создания

> Реальный пример: как сводка «здоровья проекта» перед тем, как углубляться в детали.

### Языки и участники

- Карточка **используемых языков**.
- Карточка **профилей участников** с ключевой информацией.
- Быстро видно, кто вносит вклад и над чем.

## История и анализ коммитов

### График активности

- График коммитов с выбором периода:
  - последние 7 дней
  - последние 30 дней

### Поиск по хешу коммита

- Прямой поиск по хешу коммита.
- Расширенный выбор периода:
  - 7 дней
  - 30 дней
  - 6 месяцев
  - 12 месяцев

### Детали выбранного коммита

Когда выбран коммит:

- Сообщение и описание
- Количество добавленных / удалённых строк
- Дата и время
- Количество изменённых файлов
- Кнопка перехода на GitHub

> Реальный пример: ты кликаешь по коммиту, как по строке в банковской выписке, и видишь ровно что произошло.

## Система patch notes

- Добавлена встроенная **система patch notes**.
- Поддержка markdown и изображений.
- Понятная история обновлений (как то, что пользователь читает прямо сейчас).

> Реальный пример: как changelog в SaaS-продуктах — ты знаешь, что изменилось, когда и почему.
